## 已完成

### ✅ 对象层脊柱 — `polyagents/objects.py`
- 5 个 frozen 对象 `Market / Hypothesis / Strategy / Position / Portfolio`,共享 `FO` 契约(id / version / snapshot_id / state / owner / lineage / eval_summary)。
- 状态机 `draft → lab → paper → live → archived`;`archived` 唯一终态,**无后向边**。

- `EvalSummary` 数据契约(n / brier_delta / brier_delta_ci / ece / beats_market / sample_adequate)。
- 零新依赖,不碰现有引擎。

## 待办(按依赖排序)

### ⬜ #1 对象落库 — 扩展 `storage/db.py`
- 新增 4 张表(PRD 第九节):`objects` / `forecasts` / `evaluations` / `promotion_events`。
- 对象用**单表 JSON 列**(`payload_json`),不学 v0.1 的 17 表。
- CRUD:`save_object` / `get_object` / `list_objects(type, state)` / `record_promotion`。
- 验收:对象写入后能读回、状态/version 一致;promote 在 `promotion_events` 留痕。
- **风险:低**。

### ⬜ #2 评估账本 + bootstrap CI — `evaluation/ledger.py`(**优先 / 瓶颈**)
- 现有 `evaluate.py` 只有**点估计,没有 CI** —— 门2/门3 的硬前提缺失,这是 feedback1 最核心批评。
- 落 `forecasts`(p_raw / p_cal / p_market + model_version / prompt_version / calibrator_id / prediction_time)。
- 实现 bootstrap CI(默认 1000 次重采样,可配)→ 产出 `EvalSummary`(brier_delta_ci、beats_market=CI 上界<0)。
- 生成 `EvaluationReport` 挂到 Hypothesis。
- 验收:同一组已结算市场,model vs market baseline 带 95% CI;`beats_market` 只在 CI 排除 0 时为真。
- **风险:中**(真硬骨头)。

### ⬜ #3 Lab 历史回放 — `lab/backtest.py`
- `BacktestRunner.run(hypothesis, time_window)`:取 window 内 settled 市场 → 重建 prediction_time 的 snapshot → 跑 signal 得 p_raw → 校准得 p_cal → 对比 p_market → bootstrap CI → `EvalSummary` 挂回 hypothesis。
- 复用现有 `evaluate.py` / `agents/`。
- **严格 PIT**:重建 snapshot 时过滤 `available_at <= prediction_time`。
- **风险:中**(依赖 PIT 数据是否齐备,见下「现实校准」)。

### ⬜ #4 Web 接一道门(Ask → Hypothesis)

- 新 MCP 工具:`create_hypothesis` / `run_backtest` / `get_report`。
- 改造现有 `web/`(对象感知的 chat)。
- 验收:跑通 PRD 第七节用户故事。
- **风险:低**(UI 改造)。

### ⬜ #5 AgentSession 薄层 — `runtime/session.py`(M0 后段)

## 现实校准 / 风险

- **PIT 地基是否漏水**:#3 历史回放严格 PIT,取决于现有 SQLite(`candles` / `trades` / `collections`)**有没有 `available_at` 时间戳**。若没有,门5 的「构造未来信息被拒」测试会暴露采集层就在泄漏(M1 kill criteria)。**建议在写 #2/#3 前先花十分钟查现有表够不够支撑 PIT 回放。**
- **样本量**:加密类/小众类已结算市场 n 偏小,bootstrap CI 会很宽 → 门2 大概率「方向正向但样本不足」。这是真实信号,不是 bug,如实报告。
- **两周 ~1200 行**:对象层 + 落库 + ledger 扎实;backtest 的工作量随 PIT 数据完备度浮动。
