# 策略方案:F1 安全车市场 · 区间对冲锁利

> 一句话:利用 Polymarket "本站会不会出安全车" 市场在**赛前(练习/排位)剧烈波动**的特性,
> **低点买 YES、高点补等量 NO**,把价差锁成利润。不是无风险套利,是**波动捕捉 + 锁利**。

---

## 1. 策略机制(为什么能赚)

Polymarket 一个市场有 YES / NO 两种份额,各在结算时赔付 $1;任一时刻 **YES价 + NO价 ≈ 1**。

- 在价格**低点**(如 YES=0.40)买 N 股 **YES** → 成本 0.40N
- 之后价格**涨到 0.60** 时,NO 价 = 0.40,买 N 股 **NO** → 成本 0.40N
- **两腿总成本 0.80N**,无论最终 YES 还是 NO 结算,都赔付 **N** → **净赚 0.20N,与结果无关**

**锁定利润 = 1 −(YES买价 + NO买价)**,只要两腿之和 < 1 即锁死盈利。
本质 = "买 YES 低、在更高价平掉"的低买高卖,用 YES/NO 两腿表达。

### 为什么选安全车市场
安全车是**高频事件**,且价格在赛前对练习/排位的**事故/黄红旗**剧烈重定价 → 摆幅大 → 容易凑到"两腿之和 < 1"。

## 3. 实操手册(mentor 执行)

**标的**:每站 F1 "Will there be a safety car during the [GP]?"(gamma 搜 "safety car" 可找到当前站)。

**建仓(YES 腿)**
- 时机:**FP1/FP2/排位相对干净、价格被砸到 0.40–0.55** 时买入 YES。
- 依据:基准率 ~75%,此时市价明显低于真实概率 → 单边就偏 +EV。

**对冲(NO 腿,锁利)**
- 时机:价格**因事故/黄红旗弹升到 0.70+** 时,买**等量** NO(此时 NO 便宜 ≤0.30)。
- **锁定条件:YES买价 + NO买价 < 1**。例:YES@0.45 + NO@0.30 = 0.75 → 锁定 **0.25/股**。
- 若价格只跌不涨:**不要硬凑**(NO 会变贵、两腿 >1 = 亏);此时要么扛 YES 单边(基准率高,胜面仍在),要么止损。

**离场**
- 两腿建好后可**持到结算**(已锁定),或价差进一步收敛时提前了结。

**仓位**:两腿**等量**;单站小额试水,验证滑点后再放大。

## 4. 诚实的风险与边界(务必让 mentor 知道)

1. **不是无风险套利**:利润只在**已完成对冲的那一对**上锁定;先买的 YES 腿在补 NO 之前是**方向性敞口**,需要价格朝你摆动才能完成锁利。
2. **回测是最优上限**:19% 是"完美逮住低点+随后高点"的毛值,实盘只能吃一部分。
3. **摩擦**:手续费 + 买卖价差 + 低流动性滑点会吃掉利润;这些是**小盘、深度浅**的市场,大额挂单会砸盘。
4. **样本小**:8 站,统计上偏弱;且日线数据。建议**先纸面/小额跑 3–5 站**积累真实成交数据再加码。
5. **事件风险**:比赛取消/重赛/判罚争议 → 预言机仲裁可能打乱对冲。

## 6. 结论 / 下一步

- **策略成立**:标的存在、波动足够(平均 26% 摆幅)、机制清晰、基准率有利。
- **不是无风险**:是波动捕捉 + 锁利,需纪律执行(只在两腿 <1 时锁,反向不硬凑)。
- **建议**:mentor 先在**下一站(小额/纸面)实操 3–5 站**,记录真实成交价与滑点 → 回填数据 → 再决定是否放大。系统这边可继续把 `hedge_scan` 升级成**多站批量扫**,自动列出当前所有"高摆幅、适合对冲"的事件盘。
