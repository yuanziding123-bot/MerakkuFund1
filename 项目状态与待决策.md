## 一、整体架构现状

```
L1 数据收集 ──→ L2 决策引擎 ──→ L3 执行 ──→ L4 反馈闭环
  (确定性)      信号/决策/反思    纸面/实盘+熔断   记忆/结算/反思
     │                                              │
  SQLite 库 + ChromaDB RAG          lessons ◄── settle ◄┘
     旁路: docs MCP(开发期 + 运行时)
```

| 层 | 内容 | 状态 |
|---|---|---|
| **L1 数据** | 微结构 / 情感 / 因子 / 官方 SDK 订单簿 / SQLite 缓存+历史 / ChromaDB RAG | ✅ |
| **L2 决策** | 信号(LLM)→ 决策(Kelly+风控,确定性)→ 反思(LLM) | ✅ |
| **L3 执行** | 端口/适配器(纸面默认 / 实盘 gated)+ 熔断 + 组合 | ✅ |
| **L4 反馈** | 决策日志 + 结算(按 token 判胜负)+ 结果反思 + P&L 报告 | ✅ |
| 旁路 | Polymarket docs MCP(开发期 `.mcp.json` + 运行时 `mcp_tools`) | ✅ |

入口:`collect()`(L1) / `analyze()`(L1+2) / `trade()`(L1+2+3) / `settle()`+`report()`(L4)。全程用真实 Claude 跑通过。

## 四、已修复的问题(记录备查)

**环境/工程**
- 本机无真 Python(只有 Store 桩,退出码 49)→ winget 装 3.12 + venv。
- 项目嵌套在父 git 仓库导致暂存失败 → 移到独立 `C:\polyagents`。
- PowerShell 坑:here-string 解析、`Remove-Item` 被沙箱拦、stderr 被当错误 → 用消息文件提交等绕开。
- qlib 无 C 编译器不能从源码编译 → 用预编译 pyqlib;`C:\qlib` 目录下本地源码会盖住装好的包。

**数据/逻辑 bug(已修)**
- `price_interval="1h"` 只回 2 根 K 线 → 它是"总跨度"非"bar 大小",改 `"1w"` 得 169 根。
- 决策用了滞后的 Gamma 快照价 → 改用实时 book mid(`effective_market_price`)。
- 结算按 YES/NO 标签判胜负错误(选手名市场两边都判 NO)→ 改为**按 token id 判定**。
- `fetch_market_by_condition` 拿不到已结算市场 → 加 `closed=true` 兜底。

**文档与现实不符(重要发现)**
- `pawelsibyl/marketlens-python` → 404;MarketLens 是商业服务(marketlens.trade)。
- `Polymarket/agents` → 已归档、用 OpenAI(非 Anthropic)、py3.9,功能与现有重叠。
- Alpha DevBox 被文档描述错(实为 Agent 托管平台)。
- Tavily 文档根本没提(从旧 repo 继承);文档要 Polyseer。
- 文档要 Redis/Qlib/ClickHouse + WebSocket;我们用 SQLite + 轮询。
